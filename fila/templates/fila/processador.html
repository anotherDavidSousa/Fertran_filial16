OST e CT-e

O processamento de PDFs (desmembrar, extrair dados, comprimir e enviar ao MinIO) é feito pelo n8n. O Django apenas recebe os dados já extraídos e a chave do ficheiro no MinIO, via API.

Endpoints (POST JSON + header X-Api-Key)

  • POST /api/n8n/ost/ — corpo JSON com campos da OST + pdf_storage_key (ex.: ost/7310980.pdf)
  • POST /api/n8n/cte/ — corpo JSON com campos do CT-e + pdf_storage_key (ex.: ctes/7310980.pdf)

Identificação OST: filial, serie, documento e nota_fiscal (lista ou string), ou numero_ost no formato 16.001.12345. Opcional: apenas_criar: true — se o registo já existir com PDF, não atualiza.

Autenticação: mesma API Key configurada em Admin → API Keys (ou JWT de utilizador).
